Connect multiple ad accounts per platform and manage them all from a single conversation.

## How it works

Your Adspirer account can connect to multiple ad accounts on each platform:

- Multiple Google Ads customer IDs
- Multiple Meta Business Manager ad accounts
- Multiple LinkedIn Campaign Manager accounts
- Multiple TikTok Ads Manager accounts

All connected accounts are marked as **active** — there's no hierarchy between them. When you connect during onboarding, you select which accounts to activate using multi-select checkboxes.

### Account resolution

Adspirer automatically resolves which account to use based on how many you have:

| Scenario | Behavior |
|----------|----------|
| **1 account** on a platform | Auto-selected — no action needed |
| **2+ active accounts** on a platform | You must pass the account ID to each tool call |

If you have multiple active accounts on a platform and don't specify which one, the tool will return your account list and ask you to pick one.

## Discovering your accounts

Start every session by listing your connected accounts:

```
List my connected ad accounts
```

Or call the tool directly:

```
list_connected_accounts()
```

This returns all active accounts with their platform, account ID, name, and status. Use it to:

- See which accounts are available before running any tool
- Match a client or business name to an account ID
- Get the exact parameter name for each platform (`customer_id`, `ad_account_id`, `advertiser_id`, `account_id`)

<Tip>
`list_connected_accounts` makes zero API calls — it reads directly from the database, so it's instant and doesn't count against platform rate limits.
</Tip>

You can also filter by platform:

```
list_connected_accounts(platform="google_ads")
```

## Targeting a specific account

When you have multiple accounts on a platform, pass the account ID to any tool call. The parameter name varies by platform:

| Platform | Parameter | Example |
|----------|-----------|---------|
| Google Ads | `customer_id` | `"123-456-7890"` |
| Meta Ads | `ad_account_id` | `"act_123456789"` |
| TikTok Ads | `advertiser_id` | `"7012345678901234"` |
| LinkedIn Ads | `account_id` | `"512345678"` |

Example:

```
Show me campaign performance for my Google Ads account 123-456-7890
```

Or in a tool call:

```
get_campaign_performance(customer_id="1234567890")
```

## For agencies

If you manage ads for multiple clients:

1. **Connect all client accounts** through OAuth during onboarding — select multiple accounts using the multi-select checkboxes
2. **Start each session** by listing accounts:
   ```
   List my connected ad accounts
   ```
3. **Reference the specific account** when calling tools:
   ```
   Show campaign performance for customer_id="1234567890"
   ```
4. **Run your workflow** — pass the account ID to each tool call in the session

<Tip>
If a user mentions a business name (e.g., "check performance for Acme Corp"), match it against the `account_name` from `list_connected_accounts` to find the right account ID.
</Tip>

## Managing active accounts

You can change which accounts are active from the Adspirer dashboard at [adspirer.ai/connections](https://adspirer.ai/connections). Use the "Activate accounts" option to add or remove accounts from your active set.

## Cross-account reports

Pull performance data from multiple accounts in one request:

```
Compare performance across all my Google Ads accounts for the last 30 days.
Show spend, conversions, and CPA for each account.
```

## FAQ

<AccordionGroup>
<Accordion title="Is there a limit to how many accounts I can connect?">
No hard limit. Connect as many ad accounts as you manage. Tool call limits apply to your Adspirer plan, not to the number of connected accounts.
</Accordion>
<Accordion title="Can different team members manage different accounts?">
Yes, with Claude Team or Enterprise. Each team member authenticates with their own Adspirer account and connects their own set of ad accounts. See the [Claude setup guide](/ai-clients/claude) for details.
</Accordion>
<Accordion title="Does listing accounts or switching cost a tool call?">
`list_connected_accounts` and `get_connections_status` are diagnostic tools that don't count against your tool call quota.
</Accordion>
<Accordion title="What happened to the primary account model?">
Previously, one account per platform was designated as "primary" and all tool calls defaulted to it. Now all accounts are treated equally as "active." If you only have one account on a platform, it's still auto-selected. If you have multiple, you specify which one to use.
</Accordion>
</AccordionGroup>
